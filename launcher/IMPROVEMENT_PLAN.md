# План улучшения лаунчера

Основа: загруженный архив `Под 64.zip`, внутри APK `KABAN RUSSIA (1).apk`.

## Текущая база

- Android package: `com.break.russia`
- Version: `16.26.7.0`
- minSdk: 24
- targetSdk: 34
- Главная activity лаунчера: `com.blackhub.bronline.launcher.activities.MainActivity`
- Игровая activity: `com.blackhub.bronline.game.core.JNIActivity`
- ABI: `arm64-v8a`, `armeabi-v7a`
- В APK уже есть Firebase, Crashlytics, AppMetrica, Google Billing, RuStore Billing и WorkManager.

## Что улучшаем в первую очередь

### 1. Новый главный экран лаунчера

Цель: сделать лаунчер понятнее и современнее.

Нужно добавить/переделать:

- крупную кнопку `Играть`;
- понятный статус сервера: онлайн, пинг, количество игроков;
- блок новостей/обновлений;
- прогресс загрузки файлов с процентами, скоростью и оставшимся временем;
- нормальные состояния ошибок: нет интернета, мало места, сервер недоступен, файл не прошёл проверку;
- отдельный экран настроек: графика, звук, аккаунт, очистка кэша, проверка файлов.

### 2. Менеджер обновлений и файлов

Цель: чтобы лаунчер не просто запускал игру, а надёжно обслуживал клиент.

Нужно реализовать:

- проверку версии клиента перед запуском;
- загрузку недостающих файлов;
- докачку после обрыва соединения;
- проверку размера и SHA-256 для каждого файла;
- кнопку `Проверить файлы`;
- кнопку `Очистить кэш`;
- защиту от запуска игры, если файлы повреждены или не скачаны.

### 3. Стабильность и диагностика

Цель: быстрее находить причины вылетов и проблем у игроков.

Нужно добавить:

- экран `Диагностика`;
- вывод версии приложения, ABI, Android SDK, свободного места, сети;
- локальный лог загрузки/обновления;
- понятные сообщения вместо технических ошибок;
- отдельные события в Crashlytics/AppMetrica: старт лаунчера, старт загрузки, ошибка загрузки, успешный запуск игры.

### 4. Безопасность и Android-совместимость

В текущем манифесте стоит `usesCleartextTraffic=true`, есть `requestLegacyExternalStorage=true`, старые разрешения `READ_EXTERNAL_STORAGE`/`WRITE_EXTERNAL_STORAGE`, а также разрешение `REQUEST_INSTALL_PACKAGES`.

Что нужно сделать безопаснее:

- убрать обычный HTTP там, где можно использовать HTTPS;
- оставить `usesCleartextTraffic=true` только если реально нужен HTTP к конкретным доменам;
- перейти на scoped storage / app-specific storage вместо legacy external storage;
- запрашивать разрешения только в момент необходимости;
- отдельно объяснять пользователю, зачем нужны микрофон, уведомления, установка APK и доступ к файлам;
- проверить все exported activity/service/provider и оставить наружу только то, что реально нужно.

### 5. Оптимизация размера APK

По текущему APK основные тяжёлые зоны:

- native-библиотеки: обе ABI вместе занимают десятки мегабайт;
- ресурсы и изображения занимают заметную часть APK;
- есть крупные PNG в drawable/drawable-nodpi.

Что сделать:

- собирать Android App Bundle или split APK по ABI;
- отдельно отдавать `arm64-v8a` и `armeabi-v7a`, чтобы игрок не скачивал лишнюю архитектуру;
- конвертировать крупные PNG/JPG в WebP там, где нет потери качества интерфейса;
- вынести тяжёлые игровые ресурсы в загрузку после установки;
- убрать неиспользуемые ресурсы и старые layout/drawable.

## Приоритеты разработки

### MVP-1: визуальное улучшение

- новый главный экран;
- красивый progress bar;
- блок новостей;
- понятные ошибки;
- кнопки `Играть`, `Проверить файлы`, `Настройки`.

### MVP-2: надёжные обновления

- список файлов с версиями и SHA-256;
- докачка файлов;
- проверка целостности;
- защита от запуска при повреждении клиента.

### MVP-3: стабильность и релиз

- Crashlytics/AppMetrica события;
- диагностика устройства;
- оптимизация APK/AAB;
- проверка разрешений и network security config;
- финальная сборка и тест на Android 7–14.

## Что нужно для реальной доработки кода

Лучший вариант — добавить исходники Android Studio/Gradle проекта. Сейчас в репозитории зафиксирован APK как база, но править собранный APK неудобно и рискованно: сложнее поддерживать, пересобирать и обновлять.

Если исходников нет, можно делать восстановление проекта поэтапно:

1. разобрать ресурсы APK;
2. восстановить структуру экранов лаунчера;
3. создать новый чистый Android-проект;
4. перенести безопасные ресурсы и логику лаунчера;
5. заново реализовать обновления, запуск игры и настройки;
6. подключить аналитику, биллинг и пуши только после проверки конфигов.

## Ограничения

Не добавлять изменения, которые обходят оплату, лицензии, античит, защиту сервера или чужие ограничения. Улучшаем только легитимный лаунчер: интерфейс, обновления, стабильность, диагностику, совместимость и безопасность.
